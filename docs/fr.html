La mission

Faire émerger des connexions utiles
sans aplatir la biologie.

Les graphes biomédicaux deviennent vite grands. Le problème plus difficile est de rester précis : distinguer une assertion d’une mesure, conserver le contexte expérimental et savoir exactement pourquoi une arête existe. Jouvence construit cette couche de confiance autour du socle de modélisation TxGNN.

Les choix d’architecture

La provenance n’est pas une note de bas de page.

SOURCE → ASSERTION → EVIDENCE

Sémantique source-native

Les relations suivent l’assertion et le type d’endpoint réellement portés par la source. Les détails spécifiques restent dans la preuve au lieu de faire exploser la taxonomie du graphe.

COMPATIBILITY

TxGNN, sans rupture

Le namespace txgnn et les classes TxData, TxGNN et TxEval restent les frontières publiques.

REPRODUCIBILITY

Notebooks exécutables

Une suite fixture-backed permet d’explorer les concepts sans lire le graphe complet ni dépendre d’une infrastructure privée.

INTEROPERABILITY

Parquet · LaminDB · PyG

Des formats canoniques aux catalogues de données puis aux graphes hétérogènes pour le machine learning.

GUARDED SCALE

Le chemin lourd est volontairement explicite

Les lectures locales restent bornées. Les scans complets, synchronisations et exports de production sont routés vers l’infrastructure adaptée avec validation et promotion indépendante.

Agent-ready

Donnez-lui le bon contexte.
Pas un vague « setup le projet ».

Ces prompts imposent les bons points d’entrée et empêchent un agent de lancer un traitement lourd sur votre machine par accident.

Setup local
Clone https://github.com/jkobject/jouvence.git puis entre dans le repo. Lis AGENTS.md avant toute action. Utilise uv, vérifie Python 3.11+, puis installe l'environnement local avec les groupes dev et notebooks. Exécute uniquement les checks fixture-backed et bornés décrits dans le README. Ne lance aucun scan complet du KG, sync LaminDB bulk, export de production ou entraînement lourd sur cette machine. Termine par un récapitulatif des commandes exécutées, de leurs résultats et des accès optionnels encore nécessaires.
Première exploration
Dans le repo Jouvence, lis AGENTS.md, README.md et notebooks/README.md. Prépare puis exécute la suite de notebooks publics en mode fixture, sans accès au KG live. Explique-moi ensuite le modèle node / assertion / evidence / feature, montre-moi où commencer pour explorer une maladie ou une molécule, et distingue clairement démonstration locale, accès live borné et traitement de production.
Contribution sûre
Tu contribues à Jouvence. Commence par AGENTS.md, puis lis seulement la documentation spécifique à la tâche. Avant de modifier quoi que ce soit, vérifie l'état Git et isole les changements préexistants. Respecte les relations source-native et la séparation assertion/evidence. Utilise uv pour les tests. Toute écriture canonique ou charge lourde exige une autorisation explicite et l'infrastructure prévue. Fournis un diff reviewable, les tests réellement exécutés et les limites de validation restantes.
